# Contributing

Thanks for contributing.

## Good first contributions
- Improve examples in docs.
- Add stack-specific starter-kit details.
- Improve migration checklists.
- Fix unclear onboarding steps.

## Issue labels maintainers should use
- `good first issue`
- `help wanted`
- `documentation`

## Contribution flow
1. Open an issue or pick an existing one.
2. Keep changes focused and small.
3. Include before/after context in PR description.
